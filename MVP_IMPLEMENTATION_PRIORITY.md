# MVP Implementation Priority

## Purpose

Prevent specification growth from outpacing live functionality.

Focus development effort on features that directly improve the Bellbrook pilot.

---

## Priority 1
### Fix Live Portal Workflow

Objectives:

- Ensure clean new-account creation
- Clear resident data on restart
- Prevent old submissions persisting in forms
- Maintain separate cover letter output
- Maintain separate community impact record output
- Ensure print and download workflows operate correctly

Success Measure:
Residents can complete multiple independent records without contamination.

---

## Priority 2
### Recipient Intelligence UI

Objectives:

- Display recipient guidance
- Display recipient capabilities and limitations
- Display recipient-specific questions
- Display expected responses
- Display next-step guidance

Success Measure:
Residents understand why they are contacting a recipient and what that recipient can realistically do.

---

## Priority 3
### Transparency Layer UI

Objectives:

- Clearly separate resident-authored content
- Clearly separate portal-generated content
- Display attribution information
- Preserve resident ownership of testimony

Success Measure:
No confusion between testimony and generated guidance.

---

## Priority 4
### Response Register UI

Objectives:

- Capture acknowledgements
- Capture responses
- Capture unresolved questions
- Track status progression

Success Measure:
Every submission can be tracked after it is sent.

---

## Priority 5
### Community Accountability Dashboard

Objectives:

- Aggregate response statistics
- Show unanswered questions
- Show unresolved matters
- Show engagement levels by recipient category

Success Measure:
Community-level accountability metrics become visible.

---

## Priority 6
### Community Cloning System

Objectives:

- Replace community name
- Replace issue description
- Replace recipient library
- Reuse framework components

Success Measure:
New rural communities can deploy the model without rebuilding the platform.

---

## Bellbrook Principle

Bellbrook remains the pilot deployment.

The objective is to build and validate the model through a real-world issue before replication elsewhere.
