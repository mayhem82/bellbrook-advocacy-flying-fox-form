# Bellbrook Advocacy Portal Implementation Roadmap

## Vision

Build a community evidence, submission, response, and accountability system that can later be cloned for other rural and remote communities.

The Bellbrook Flying Fox project is the pilot implementation.

---

## Phase 1
### Community Impact Record

Purpose:
Collect resident testimony and impacts.

Outputs:
- Community Impact Record
- Resident-authored evidence

Status:
Implemented

---

## Phase 2
### Submission Package System

Purpose:
Generate complete submission packages.

Outputs:
- Community Impact Record
- Cover Letter
- Recipient Directory

Status:
Implemented

---

## Phase 3
### Recipient Intelligence Layer

Purpose:
Translate resident evidence into recipient-specific context.

Outputs:
- Recipient Briefings
- Recipient Questions
- Recipient Guidance
- Transparency Statements

Status:
Specified

---

## Phase 4
### Response Register

Purpose:
Capture replies and preserve institutional memory.

Outputs:
- Submission Register
- Response Register
- Escalation Tracking

Status:
Specified

---

## Phase 5
### Community Accountability Dashboard

Purpose:
Aggregate responses and identify unresolved issues.

Outputs:
- Response Metrics
- Outstanding Questions
- Community Accountability Records

Example Metrics:
- Records Submitted
- Responses Received
- Questions Answered
- Questions Outstanding

Status:
Planned

---

## Phase 6
### Cloneable Community Template System

Purpose:
Allow the Bellbrook model to be reused elsewhere.

Potential Communities:
- Taylors Arm
- Willawarrin
- Smithtown
- Sherwood
- Toorooka
- Other rural and remote communities

Reusable Components:
- Community Impact Record
- Recipient Intelligence Layer
- Response Register
- Accountability Dashboard

Status:
Future Development

---

## Core Design Principles

- Simple for residents
- Transparent
- Evidence-first
- Response-focused
- Community memory preservation
- Recipient-specific translation
- Resident ownership of testimony

---

## Long-Term Model

Community Problem

→ Community Evidence

→ Structured Record

→ Recipient-Specific Translation

→ Submission

→ Response Capture

→ Public Accountability Record

→ Community Memory

→ Cloneable Community Model
