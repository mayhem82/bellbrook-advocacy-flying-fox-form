# Bellbrook Advocacy Portal — Response Register and Community Memory Specification

## Purpose

The Bellbrook Flying Fox Community Impact Record is the front door. The long-term value is the record of what was submitted, who received it, what response was given, and what remains unresolved.

This phase converts the portal from a form generator into a community memory system.

## Core Lifecycle

1. Record Created
2. Package Generated
3. Recipient Selected
4. Submission Sent
5. Acknowledgement Received
6. Response Received
7. Next Action Identified
8. Escalated / Resolved / Closed

## Response Register Fields

- Submission ID
- Resident name or anonymous reference
- Date record created
- Date sent
- Recipient category
- Specific recipient
- Method sent
  - Email
  - Post
  - Hand delivered
  - Online form
- Package type
  - Resident account only
  - Account plus cover letter
  - Account plus recipient module
  - Full submission package
- Acknowledgement received
- Acknowledgement date
- Response received
- Response date
- Response summary
- Documents returned or referenced
- Questions answered
- Questions not answered
- Next action
- Status
  - Created
  - Sent
  - Awaiting acknowledgement
  - Awaiting response
  - Responded
  - Escalated
  - Closed

## Community Accountability Metrics

The portal should eventually be able to show aggregated, non-private counts such as:

- Records created
- Records submitted
- Recipients contacted
- Acknowledgements received
- Substantive responses received
- Responses outstanding
- Issues escalated
- Issues resolved

Example:

Recipient: Kempsey Shire Council

Records Sent: 27

Acknowledged: 25

Substantive Responses: 4

Outstanding: 23

## Transparency Requirements

The system must clearly separate:

- Resident-authored content
- Portal-generated cover letters
- Portal-generated recipient modules
- Portal-generated guidance
- Recipient responses
- Community-level summaries

No portal-generated material should be presented as personal testimony unless the resident explicitly adopts it.

## Resident Experience

The resident should only need to complete the following steps:

1. Create their Bellbrook Flying Fox Community Impact Record
2. Choose a recipient
3. Review the generated package
4. Send the package
5. Record any response received

The portal handles explanation, translation, and recipient-specific context.

## Recipient Intelligence Layer

Each recipient module should explain:

- Why this recipient may matter
- What this recipient can do
- What this recipient cannot do
- Questions the package raises
- What response may be expected
- What the resident should do next

## Final Model

Community Problem

→ Community Evidence

→ Structured Record

→ Recipient-Specific Translation

→ Submission

→ Response Capture

→ Public Accountability Record

→ Community Memory
